## CRITICAL (без этих правок нельзя запускать платно)

### Fix 1: Предварительная экономика от min известной цены
**Скор: Decision value 4→7, Trust 6→7, Willingness to pay 5→7**

**Проблема:** В `Данные 1688` видны цены 28/27/26¥, но экономика показывает нули и "цена не распознана". Пользователь видит цены, а бот их игнорирует.

**Файл:** `src/providers/productImporter.ts` (строки 236-270) + `src/core/economicsCalc.ts`

**Что делать:**
```
productImporter.ts — buildNormalizedProduct():
  Если displayPrice = 0 но есть tieredPrices или skuPrices:
    displayPrice = Math.min(...все_известные_цены)
    pricing.isEstimated = true
    pricing.estimateSource = 'min_tiered' | 'min_sku' | 'promo'

economicsCalc.ts:
  Если isEstimated = true:
    Считать всю экономику как обычно
    Добавить флаг isSyntheticPrice = true
    В breakdown добавить: "⚠️ Цена предварительная (мин. из прайс-листа)"
```

**Результат:** Пользователь ВСЕГДА получает хотя бы предварительный расчёт вместо нулей.

### Fix 2: Ручной ввод веса + категорийный дефолт
**Скор: Decision value 7→9, Time saving 7→8**

**Проблема:** Без веса карго/ROI/бюджет = 0. Нет пути ввести вес. В `tariffs.ts` уже есть механизм inline-ввода через Redis — нужно добавить вес.

**Файлы:** `src/bot/handlers/tariffs.ts` + `src/core/economicsCalc.ts` + `src/core/messageBuilder.ts`

**Что делать:**
```
1. tariffs.ts — добавить поле weightKg:
   Кнопка "⚖️ Указать вес" в разделе Экономика (рядом с "Изменить параметры")
   При клике: Redis tariff_pending:{chatId} = { field: 'weightKg' }
   Пользователь вводит число → сохраняется в userSettings
   → пересчёт экономики → обновление сообщения

2. economicsCalc.ts — категорийный дефолт:
   Если weightKg = 0 и есть intelligence.category:
     одежда/текстиль → 0.3 кг (дефолт)
     обувь → 0.8 кг
     электроника → 0.5 кг
     аксессуары → 0.2 кг
   Пометить: weightSource = 'category_default'
   В breakdown: "⚖️ Вес оценочный (~300г, категория: одежда)"

3. messageBuilder.ts — в секции Экономика:
   Показать кнопку "⚖️ Указать вес" если weightMissing
   Показать дефолтный вес если используется
```

**Результат:** Экономика считается ВСЕГДА — с реальным, введённым или дефолтным весом.
