# CardZip — release blockers cleanup patch

Исправлены блокеры, найденные после проверки Telegram UX и файлов.

## Что изменено

1. **Callback-кнопки теперь несут analysisId/jobId в формате с контекстом**
   - `supplier_questions:{jobId}`
   - `package:{jobId}`
   - `product_details:{jobId}`
   - `back_main:{jobId}`
   - `package_zip:{jobId}`

   Для обратной совместимости старые callback-и с `_` тоже поддерживаются.

2. **Починен сценарий “Вопросы поставщику”**
   - Основные кнопки больше не используют generic `supplier_questions`.
   - Handler ищет анализ по jobId, а не по текущему session state.
   - Generic callback без id не открывает случайный последний товар.

3. **Убран двойной счётчик анализов**
   - Главный отчёт больше не вставляет строку `Осталось анализов: N`.
   - `applyCreditsLine()` удаляет старые варианты счётчика и добавляет только одну строку:
     `📦 Осталось: N анализов`.

4. **Исправлена опечатка/латиница в русском слове**
   - Добавлен sanitizer для `поставщpику` → `поставщику` и похожих смешанных русско-латинских случаев.

5. **Русские имена файлов в ZIP**
   - `00_Инструкция.txt`
   - `01_Вопросы_поставщику.txt`
   - `02_ТЗ_байеру.md`
   - `03_ТЗ_карго.md`
   - `04_Чеклист_образца.md`
   - `05_SEO_черновик.md`
   - `06_Фото_товара.zip`

6. **Экран “Закупочный пакет” стал понятнее**
   - Русские имена файлов показаны прямо в интерфейсе.
   - Добавлено описание, кому и зачем нужен каждый файл.
   - Кнопки: вопросы поставщику, скачать ZIP, данные товара, назад к отчёту.

7. **Общий dedup для документов**
   - Добавлен `dedupNormalizedList()`.
   - Нормализуются близкие формулировки: вес с упаковкой / точный вес / вес одной единицы с упаковкой.
   - Добавлен markdown line dedup внутри валидатора документов.

8. **Category-specific выводы**
   - Балаклава: состав ткани, размеры, посадка, УФ-защита, дыхание, швы, растяжимость.
   - Зонт: механизм, спицы, купол, чехол, размер, UPF.
   - Обувь: стелька, запах EVA/PU, размерность, литьё/склейка, упаковка.
   - Маска для сна и мини-стиральная машина тоже получили более конкретный verdict.

9. **Балаклава распознаётся как clothing-подтип**
   - Добавлены ключи: балаклава, подшлемник, face mask, 防晒面罩 и др.
   - Для балаклавы применяются отдельные supplier questions / sample checks / SEO.

10. **SEO для балаклавы улучшен**
    - Название: `Балаклава защитная от солнца и ветра для велосипеда и активного отдыха`.
    - Описание стало товарным, а не техническим черновиком.
    - Буллетов ровно 5.
    - UPF/УФ-защита не утверждается как факт без подтверждения.
