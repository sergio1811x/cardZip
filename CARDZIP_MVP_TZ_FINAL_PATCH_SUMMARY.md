# CardZip MVP — финальный патч под упрощённое ТЗ

Основа: `cardzip_final_tz_done_patch.zip`.

Цель нового ТЗ: упростить CardZip до продукта `ссылка 1688 → закупочный пакет`, без CRM-логики, WB-аналитики и отдельного главного экрана “Дальнейший план”.

## Что изменено

### 1. Верхнее меню после отчёта

После главного отчёта теперь показываются только 4 действия:

- `💬 Вопросы поставщику`
- `📁 Закупочный пакет`
- `📦 Данные товара`
- `🔄 Новый товар`

С первого уровня убраны:

- `🚀 Дальнейший план`
- `📥 Ответ поставщика`
- `⚖️ Указать вес`
- `💸 Себестоимость`
- `⚠️ Риски`
- `🧪 Образец`
- старое `📁 Материалы`

### 2. Главный отчёт упрощён

Главный отчёт перестроен под MVP-формат:

- товар;
- цена;
- выбранный SKU;
- MOQ;
- SKU / цвета / размеры или параметры;
- материал;
- вес;
- статус без баллов;
- что уточнить;
- предварительная себестоимость;
- закупочный пакет;
- короткий вывод;
- CTA на вопросы поставщику и ZIP.

Убрано из пользовательского отчёта:

- `Готовность 70/100` и любые баллы готовности;
- CTA на `🚀 Дальнейший план`;
- ROI-формулировки;
- category default weight;
- длинный список материалов.

### 3. “Дальнейший план” убран как основной сценарий

Основной UX теперь:

1. пользователь отправляет ссылку;
2. получает короткий отчёт;
3. открывает вопросы поставщику;
4. скачивает закупочный пакет;
5. при необходимости обновляет пакет по ответу поставщика.

Legacy callback `proc_plan_*` оставлен только как безопасная совместимость со старыми кнопками, но он больше не является главным экраном и не выводится в новом меню.

### 4. “Ответ поставщика” переименован

Пользовательский термин заменён на:

`📥 Обновить пакет по ответу`

Экран объясняет, что бот извлечёт:

- цену выбранного SKU;
- вес с упаковкой;
- габариты упаковки;
- MOQ;
- комплектацию;
- материал;
- сроки;
- фото/видео/документы.

После обработки показывается `✅ Закупочный пакет обновлён` и предложение скачать обновлённый ZIP.

### 5. Закупочный пакет сокращён

ZIP теперь содержит английские имена файлов:

- `supplier_questions.txt`
- `buyer_brief.md`
- `cargo_brief.md`
- `sample_checklist.md`
- `seo_draft.md`
- `README.txt`
- `photos.zip`

Отдельные документы удалены из MVP-пакета:

- `risk_checklist.md`
- `sample_plan.md`
- `infographic_brief.md`

### 6. Объединены риск и образец

`risk_checklist` + `sample_plan` объединены в:

`sample_checklist.md`

Структура:

- до заказа образца;
- какой SKU взять;
- что проверить на образце;
- что измерить;
- какие фото сделать;
- красные флаги;
- решение после образца.

### 7. Инфографика перенесена в SEO

Отдельный файл инфографики больше не создаётся.

Идеи для инфографики добавлены в конец `seo_draft.md` в раздел:

`## Идеи для инфографики`

### 8. README добавлен в ZIP

`README.txt` объясняет:

- что внутри пакета;
- для чего каждый файл;
- рекомендуемый порядок работы.

### 9. CN-вопросы защищены validator’ом

Для китайского блока добавлена защита:

- если в CN-вопросах есть кириллица;
- если есть `file://`;
- если есть вложенная сломанная нумерация;
- если текст выглядит как языковая мешанина;

CN-блок скрывается и пользователь получает fallback:

`Китайская версия не сформирована — используйте русскую версию или переведите через байера.`

### 10. SKU normalizer сохранён

Для зонта непонятный SKU `40` не превращается в `Размер: 40`.

Показывается:

`Параметр SKU: 40 — уточнить у поставщика`

В вопросы поставщику добавляется вопрос о значении параметра.

### 11. Навигация

На вложенных экранах есть возврат:

- `⬅️ Назад`
- `🏠 К отчёту`
- `📁 Закупочный пакет`
- `🔄 Новый товар`

Тупиковые экраны убраны.

### 12. Ошибки

Пользовательский текст `Внутренняя ошибка` не используется.

Fallback:

`⚠️ Не удалось открыть раздел. Данные анализа сохранены. Попробуйте вернуться к отчёту или открыть пакет ещё раз.`

В логах остаётся `[ui-handler-error]` с action/userId/productId/state/error.

- нет `category default weight` в главном отчёте;
- ZIP создаётся с английскими именами файлов.

## Что осталось как legacy/internal

- Некоторые внутренние модули экономики и QA всё ещё знают про ROI как технический термин, но пользовательский MVP-flow его не выводит.
- Legacy callbacks оставлены, чтобы старые inline-кнопки не ломали пользователя, но они ведут в новый пакетный сценарий.
- Полный `npm run typecheck && npm test && npm run build` нужно выполнить в реальном окружении проекта после `npm install`, потому что sandbox-проверка не заменяет production build.
