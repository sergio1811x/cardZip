# CardZip MVP — cleanup patch after paid-test audit

Основа: `cardzip_mvp_final_tz_patch.zip`.

## Что исправлено

1. **Баг “Товар не найден” по кнопкам**
   - Для `supplier_questions_{jobId}` и `supplier_confirm_{jobId}` поиск анализа больше не фильтруется по статусам `done/sent`.
   - Это убирает race-condition, когда свежий отчёт уже показан в Telegram, но статус job ещё не успел стать `sent`.
   - Generic callbacks без `jobId` больше не открывают “последний товар”, чтобы не подменять контекст.
   - Пользовательский fallback заменён на: `⚠️ Не удалось открыть раздел. Но анализ сохранён...`.

2. **Контекст в inline-кнопках**
   - Основные кнопки работают только с `jobId`: `supplier_questions_{jobId}`, `materials_{jobId}`, `product_detail_{jobId}`, `supplier_confirm_{jobId}`.
   - Кнопки RU/CN вопросов также привязаны к анализу: `sq_ru_{jobId}`, `sq_cn_{jobId}`.

3. **Русификация поставщика**
   - `seller → продавец`
   - `merchant → проверенный продавец`
   - `factory → фабрика`
   - Используется в главном отчёте, деталях товара и ТЗ байеру.

4. **Материал зонта очищен**
   - Для зонтов мусорные материалы вроде `понтиж` не выводятся в main report.
   - Новый текст: `ткань купола, железо/сплав — подтвердить у поставщика`.

5. **SKU зонта стали чище**
   - В главном отчёте больше нет повторов `Параметр SKU: ... — уточнить у поставщика` для каждого значения.
   - Формат main report:
     - `SKU: N вариантов · цвет × параметр SKU`
     - `Параметры: 8 / 16 / 40 / 120 — значение нужно уточнить`

6. **Дедуп вопросов поставщику**
   - Для зонта максимум 8 вопросов.
   - Вопрос по параметрам SKU объединён в один.
   - Дубли веса убраны.
   - CN-блок использует десятичную точку: `12.39 元`, не `12,39元`.

7. **Buyer brief короче**
   - Убран отдельный длинный блок с готовыми вопросами поставщику.
   - `Что подтвердить у поставщика` ограничено до 10 строк.
   - `Что проверить на образце` ограничено до 8 строк.

8. **Sample checklist очищен**
   - Для зонта отдельный чистый сценарий:
     - до заказа образца;
     - какой SKU взять;
     - что проверить на образце;
     - какие фото сделать;
     - красные флаги;
     - решение после образца.
   - Убраны повторы веса/размеров.

9. **SEO очищен**
   - Название зонта улучшено: `Зонт автоматический складной с крючком и чехлом`.
   - Таблица характеристик для зонта стала ручной и без дублей `Тип/material`.
   - UPF50+ остаётся только как заявленное свойство, не как факт/claim.

10. **Экран “Закупочный пакет” стал понятнее**
   - Добавлены русские подписи + стабильные английские имена файлов.
   - Кнопки экрана:
     - `⬇️ Скачать ZIP`
     - `💬 Вопросы поставщику`
     - `📦 Данные товара`
     - `⬅️ Назад`

11. **README.txt переписан**
   - Добавлен порядок работы с конкретными файлами:
     - supplier_questions.txt поставщику;
     - buyer_brief.md байеру;
     - cargo_brief.md карго;
     - sample_checklist.md после образца;
     - seo_draft.md как черновик карточки.
